## 2024-05-14 - Process watcher string properties allocation & get process HashSet
**Learning:** `ProcessWatcherService` constantly polls processes and accesses `NormalizedProcessName` and `NormalizedResolvedTarget`. If these are computed property getters (that allocate a trimmed and lower-cased string on each access), this leads to continuous GC pressure, particularly in background services polling frequently. Additionally, populating a default-sized `HashSet` with `Process.GetProcesses()` (which returns an array of known size) forces multiple dynamic array resize allocations internally.
**Action:** Always memoize normalized strings in property setters rather than computing them on the fly if they are accessed repeatedly in polling loops. When creating a `HashSet` or `List` from an array of known length, pass the length as the initial capacity constructor parameter to avoid internal reallocation overhead. Also use `for` loop over arrays when in hot path to prevent `IEnumerator` allocation overhead from `foreach`.
