# DisplayPilot screenshots

Replace the placeholder PNGs in this folder with real captures before a marketing release. The placeholders are colored rectangles with labels so the GitHub README renders correctly until you drop in final shots.

## How to capture (Windows)

1. Open DisplayPilot and navigate to the screen you want.
2. Press **Win+Shift+S** to open Snipping Tool (region capture).
3. Draw a rectangle around the window or panel.
4. Click the preview notification, then **Save as** — use the exact filename below.
5. Save into this folder (`docs/screenshots/`).

## Required files

| Filename | What to capture |
| --- | --- |
| `panel-displays.png` | Flyout panel — **Displays** tab: monitor cards, arrangement map, resolution dropdowns |
| `panel-advanced.png` | Flyout panel — **Advanced** tab (if visible) or secondary panel view with map + status |
| `settings-profiles.png` | Settings window — auto-swap profile list and editor (sample Steam → game rule) |
| `wizard.png` | First-run setup wizard on the monitor-selection or finish step |

## Tips

- Use the same dark theme / monitor layout for a consistent README look.
- Crop tightly around the window; 1200–1600 px wide is usually enough.
- PNG format only (GitHub renders these inline in `README.md`).

After saving real screenshots, commit them and push — no code changes needed.
